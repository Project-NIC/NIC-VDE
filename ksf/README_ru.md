# NIC-KSF — Kolmogorov Shannon Feistel

## Библиотека шифрования для встраиваемых устройств

[![License: MIT](https://img.shields.io/badge/License-MIT-red.svg)](https://opensource.org/licenses/MIT)

---

## Что такое KSF?

NIC-KSF — это лёгкая симметричная библиотека шифрования, построенная на блочном шифре **SPECK-128** в режиме **CTR (счётчик)**. Она разработана для микроконтроллеров с ограниченными ресурсами, таких как ATmega328, где оперативная память и Flash-память дефицитны, а вычислительные затраты должны быть минимальными.

Библиотека имеет единственную чёткую ответственность: **зашифровать или расшифровать блок данных с помощью 128-битного ключа**. Всё управление ключами, их формирование, управление сессиями и протокольная логика намеренно делегируются вышестоящим уровням.

---

## Возможности

- Блочный шифр SPECK-128/128
- Режим CTR — шифрование и дешифрование являются идентичными операциями (XOR ключевого потока)
- Операция на месте — дополнительный буфер не требуется
- Поддерживает любую длину данных от 1 до 255 байт
- Отсутствие динамического выделения памяти (`malloc`)
- Никаких зависимостей, кроме `<stdint.h>` и `<string.h>`
- Совместимость с AVR (ATmega328) и стандартными компиляторами C99
- Два варианта реализации — см. ниже

## Варианты реализации

Оба варианта используют одинаковый интерфейс (`nic_ksf.h`) и дают идентичные результаты.

| Файл | Описание |
|---|---|
| `nic_ksf_32.c` | Ручная 32-битная арифметика — для старых версий `avr-gcc` |
| `nic_ksf_64.c` | Нативный `uint64_t` — для новых версий `avr-gcc` и PC |

Новые версии `avr-gcc` (Arduino IDE) могут транслировать нативные 64-битные операции в очень эффективную последовательность инструкций `add` / `adc`. Рекомендуем протестировать оба варианта и выбрать тот, который даёт меньший или более быстрый код для вашего проекта.

## Модель безопасности

NIC-KSF является **чистой криптографической примитивой**. Она не управляет ключами, сессиями или счётчиками пакетов.

**Вызывающая сторона несёт ответственность за то, чтобы каждый вызов `ksf_encrypt` получал уникальный 128-битный ключ.**

Если один и тот же ключ будет использован для двух разных пакетов, злоумышленник сможет выполнить XOR перехваченных шифртекстов и ключевой поток взаимно уничтожится. Обеспечение уникальности ключа является полной ответственностью вызывающего уровня.

## Интерфейс (API)

```c
#include "nic_ksf.h"

/* Шифрует данные на месте с использованием 128-битного ключа.
 * key  : 16 байт (128 бит) — подготавливается вышестоящим уровнем
 * data : указатель на буфер (перезаписывается зашифрованным результатом)
 * len  : количество байт (1–255)
 */
void ksf_encrypt(const uint8_t key[KSF_KEY_SIZE], uint8_t *data, uint8_t len);

/* Расшифровывает данные на месте. В режиме CTR идентично ksf_encrypt. */
void ksf_decrypt(const uint8_t key[KSF_KEY_SIZE], uint8_t *data, uint8_t len);
```

---

## Пример использования

```c
#include "nic_ksf.h"

/* 128-битный ключ, подготовленный вызывающим уровнем */
uint8_t key[16] = { /* ... 16 байт ... */ };

/* Данные для шифрования */
uint8_t payload[20] = { /* ... данные ... */ };

/* Шифрование на месте */
ksf_encrypt(key, payload, sizeof(payload));

/* ... передача ... */

/* Дешифрование на стороне получателя */
ksf_decrypt(key, payload, sizeof(payload));
```

## Сборка

### PC / Linux

```bash
# 32-битный вариант
gcc -std=c99 -Wall -Isrc -o test_ksf tests/test_ksf.c src/nic_ksf_32.c

# 64-битный вариант
gcc -std=c99 -Wall -Isrc -o test_ksf tests/test_ksf.c src/nic_ksf_64.c

./test_ksf
```

Или просто запустите `make` (соберёт и запустит оба варианта).

### AVR / ATmega328

```bash
avr-gcc -std=c99 -mmcu=atmega328p -Os -Isrc -o nic_ksf.elf src/nic_ksf_32.c
```
