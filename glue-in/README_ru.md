```
   строка датчика / пакет из линии ──▶ [ опц. NIC-DMD ] ──▶ контейнер NIC-MLA
                                                                    │
                                                                    ▼
                                       NIC-VDE  (просмотр / экспорт, только чтение)
```

> **Прочтите сначала это.** Склейка — *ваша, формируйте её сами.* Связать эти
> библиотеки можно многими правильными способами, и лучший зависит от вашего
> устройства, канала и того, что вы хотите делать с данными. Поэтому этот
> репозиторий — **рабочий пример плюс каталог вариантов**, а не фреймворк,
> который обязательно принимать. Долгосрочная ценность здесь —
> **[справочник согласования библиотек](#справочник-согласования-библиотек)**:
> небольшой набор стыков, где библиотеки должны договориться, записанный один
> раз, чтобы каждый в нём разобрался. Направление чтения / экспорта — это
> родственный проект **NIC-GLUE-OUT**; **NIC-VDE** — это просмотрщик.

## Справочник согласования библиотек

Библиотеки NIC намеренно *тупые и независимые*: MLA хранит непрозрачные байты,
DMD кодирует пакеты фиксированной ширины, VDE просматривает файлы, KSF
преобразует байты. Ни одна не знает об остальных. Склейка — это любой код,
который выравнивает эти стыки. Их всего горстка, и сделать их правильно — это
вся работа:

| Стык | Что предоставляет каждая сторона | Как они стыкуются |
|---|---|---|
| **Бит `compressed` + `kf_back`** | лог MLA v1.1 несёт 1-байтовый `flags` (бит 7 = `compressed`, биты 0–6 = `kf_back`), но не интерпретирует его; *какой* кодек применён, живёт в заголовке самого блока данных (DMD байт 0), никогда в MLA | склейка ставит бит `compressed` (`False` для сырых строк, `True` для вывода DMD) и `kf_back`; виды записи — **raw** (несжатая), **keyframe** (сжатая, `kf_back == 0`), **delta** (сжатая, `kf_back > 0`) |
| **Keyframe** | keyframe DMD = номер выборки `0` (3-битовое поле; значение `7` зарезервировано под версию протокола) | склейка читает это обратно из DMD-блоба (`blob[0] & 0x07 == 0` ⇒ keyframe = выборка DMD 0) и помечает запись соответственно |
| **Расстояние до keyframe** | лог MLA имеет поле `kf_back`, которое он лишь несёт; читателю нужно найти свой keyframe | склейка ставит `kf_back` = число записей назад к своему keyframe (`0` на keyframe) |
| **Подсказка частоты keyframe** | префикс MLA имеет `keyframe_intv` (только метаданные); частота DMD внутренняя (`DMD_KEYFRAME_EVERY`) | дефолт библиотеки `0`; склейка подставляет частоту DMD, чтобы вызывающему не набирать её вручную (переопределяемо) |
| **Субсекундный тик** | лог MLA несёт поле `subsec` (субсекундный тик / индекс выборки) рядом с 4-байтовым `timestamp` | склейка пробрасывает `subsec` в `log_raw` / `CompressedChannel.log` |
| **Ширина пакета** | DMD требует, чтобы каждый пакет в потоке имел *одинаковую* ширину (дельта) | ширина принадлежит **каналу** (4..255 Б), проверяется при каждом `log()`; разные каналы могут отличаться |
| **Идентичность потока** | идентичность потока в файле *и есть* его индекс станции MLA; MLA не нужен никакой другой тег на запись | читатель различает потоки по станции и читает `kf_back`, чтобы найти keyframe каждого потока; один без состояния компрессор DMD + N крошечных контекстов на поток (`ChannelBank`) держат дельты в порядке |
| **Ротация → keyframe** | MLA v1.1 (2b) выносит наружу событие ротации, чтобы каждый ротированный файл декодировался независимо | склейка при перекате форсирует keyframe: `MlaArchive(dir, on_rotate=bank.on_rotate)` (или проверьте `arch.will_rotate(n)` перед кодированием) → `ChannelBank.on_rotate` вызывает `reset_all()`, так что первая запись каждого потока в новом файле — keyframe (для RAW-данных не имеет значения) |
| **Станция** | лог MLA хранит 1-байтовый *индекс* станции (1..255), реальные номера живут в таблице станций в префиксе | склейка / `MlaStationTable` владеет соответствием индекс ↔ регион/номер |
| **Время** | лог MLA имеет выделенный 4-байтовый `timestamp`; поле схемы `log("datetime")` (пресет `4 Б unix_s`) его *описывает* | время живёт в заголовке лога, **не** дублируется в блоке данных — см. [варианты времени](#1-откуда-берётся-метка-времени) |
| **Раскладка полей** | схема разделяет поля `log(...)` (заголовок) и поля `data(...)` (полезная нагрузка); `mla_decode_payload` распаковывает блок | разделение log vs. data *и есть* карта «что идёт в заголовок» vs. «что остаётся в блоке» |
| **Целостность** | MLA покрывает лог-запись (и опционально блок данных) через CRC16 | выберите `MLA_CRC_FULL` (рекомендуется), `MLA_CRC_DATA` или `MLA_CRC_NONE` при форматировании |

Если ваша собственная склейка уважает эту таблицу, ваши файлы проходят туда и
обратно через NIC-VDE и NIC-GLUE-OUT независимо от того, как устроено остальное.

## Что даёт пример

Намеренно маленький даталоггер поверх одного контейнера MLA:

- **`GlueLogger`** — `log_raw()` / `log_event()`: взять строку, сохранить
  строку. Повседневный случай; работает для любого числа станций.
- **`CompressedChannel`** — `open_compressed_channel(station, pkt_len)`, затем
  `.log(ts, row)`: опциональное сжатие NIC-DMD для **одного потока фиксированной
  ширины**, с автоматически заполненным битом `compressed` / `kf_back`.
- **`ChannelBank`** — `open()` / `log()` / `reset_all()` / `on_rotate()`: один
  без состояния компрессор DMD + N крошечных контекстов на поток, по одному
  `CompressedChannel` на индекс станции MLA. Подключите
  `MlaArchive(dir, on_rotate=bank.on_rotate)`, чтобы каждый ротированный файл
  начинал каждый поток с keyframe.

```python
from nic_glue_in import GlueLogger, MlaSchemaBuilder, MlaStationTable

schema = MlaSchemaBuilder(); schema.log("datetime")          # описывает метку времени лога
for n in ("temp", "humidity"): schema.data(n, unit="raw", width=2)
stations = MlaStationTable(); stations.station(region=55, number=25000)

with GlueLogger("out.mla", schema_table=schema.table(),
                station_table=stations.table()) as log:
    log.log_raw(ts, station=1, data=row_bytes)            # классический путь (raw)
    log.log_event(ts, station=1, text="PING")             # просто несжатая запись

    ch = log.open_compressed_channel(station=1, pkt_len=4) # опциональное сжатие
    ch.log(ts, row_bytes)                                  # → compressed, kf_back (keyframe/delta)
```

```bash
python3 examples/weather_datalogger.py     # пишет weather_raw.mla + weather_dmd.mla
python3 tests/test_glue.py                  # или: pytest tests/
```

## Варианты проектирования и как это сделать

Это *возможности*, а не требования — выбирайте подходящее. Пример реализует
самый простой вариант каждого; остальное намечено, чтобы вы расширили.

### 1. Откуда берётся метка времени

Лог-запись MLA имеет выделенный 4-байтовый `timestamp`, отдельный от
непрозрачного блока данных, и поле схемы `log("datetime")` его описывает. Так
что время принадлежит **заголовку лога**, никогда не дублируется в данных. Как
оно туда попадёт — ваш выбор:

- **(a) Собственные часы склейки (RTC / время приёма).** Самое простое: склейка
  ставит каждой записи время, когда она её *приняла / залогировала*, из RTC
  устройства. Пакет несёт только данные датчиков. Так делает пример —
  `timestamp` является аргументом `log_raw` / `Channel.log`.
- **(b) Извлечённое из заголовка пакета.** Пакет из линии сам несёт время как
  заголовок (напр. `[datetime 4 Б unix_s][датчики …]`). При приёме склейка
  отрезает заголовок — **ширины log-полей схемы говорят, где он** — пишет его
  в `log.timestamp`, а оставшиеся байты датчиков сохраняет как блок. «Заголовок
  переезжает в лог.» DMD ничего не знает о времени; смещение знает *схема*.
- **(c) Передано вызывающим.** Слой выше, который уже знает достоверное время,
  передаёт его напрямую.

> Процедура для (b), приём из линии:
> `recv(blob)` → `DmdDecoder.decompress(blob)` → `packet` →
> `t = int.from_bytes(packet[:4], "little")` → `data = packet[4:]` →
> `MlaCore.append(t, station, data, compressed=…, kf_back=…)`.

### 2. Сжато на хранении или только в линии?

1-байтовый заголовок DMD и свойство «никогда не увеличивает больше чем на 1 Б,
никогда не теряет данные» делают хранение в сжатом виде безопасным. Два подхода:

- **Хранить RAW (распакованным).** Если вы приняли сжатый пакет, распакуйте его
  при приёме и сохраните байты датчиков дословно (запись вида **raw** — бит
  `compressed` остаётся сброшен). Читателю не нужен кодек; VDE декодирует прямо
  из схемы. Стоит места, покупает простоту.
- **Хранить сжато (запись **keyframe**, затем **delta**).** Оставьте DMD-блоб в
  блоке данных (бит `compressed` установлен; `kf_back == 0` помечает keyframe,
  `kf_back > 0` — дельту). Меньше файлы. Цена — **произвольный доступ**: поскольку каждый
  дельта-пакет относителен к предыдущему, чтобы открыть запись *i*, нужно
  проиграть поток от его keyframe вперёд — ровно для этого и нужен `kf_back`
  (говорит читателю, как далеко назад сидит keyframe). Для одного канала это
  один маленький буфер «предыдущей выборки» и проход от keyframe.

### 3. Несколько потоков

`CompressedChannel` — это один поток DMD = один индекс станции + одна
фиксированная ширина. Модель — **один без состояния компрессор DMD + N
крошечных контекстов на поток**, ровно это и предоставляет `ChannelBank`: он
управляет несколькими `CompressedChannel`, по одному на индекс станции MLA
(`open` / `log` / `reset_all` / `on_rotate`). Можно открыть много (до 255), но
дельта даёт выигрыш только *внутри* потока, так что сжатие десятков независимых
станций в основном лишь стоит RAM (по одному буферу предыдущей выборки на
каждую).

При ротации файла (NIC-MLA 2b) подключите `MlaArchive(dir, on_rotate=bank.on_rotate)`
(или проверьте `arch.will_rotate(n)` перед кодированием): `ChannelBank.on_rotate`
вызывает `reset_all()`, так что первая запись каждого потока в новом файле —
keyframe, и каждый ротированный файл декодируется независимо. Пример сжимает
одну станцию, чтобы показать, что это работает; всё остальное логируется raw.

### 4. Шифрование (NIC-KSF)

KSF намеренно **не** в пути хранения — хранить шифртекст в контейнере — это
неправильный слой (конфиденциальность на хранении оставьте доверенной
платформе). Его место — **транспортный** путь: отправитель шифрует (опционально
сжатый) пакет перед передачей, получатель расшифровывает перед приёмом. Ключом
владеют оба конца; контейнер его никогда не видит.

> Порядок в линии (отправитель): `упаковать строку → [сжатие DMD] → [шифрование KSF] → передать`.
> Получатель зеркалит: `приём → [расшифровка KSF] → [распаковка DMD] → сохранить`.
> Внимание: DMD считает шифрованные байты случайными и хранит их как RAW (+1 Б),
> поэтому **сжимайте до шифрования**, никогда не наоборот.

## Датологгер (мульти-профиль)

Запись нескольких типов станций в один `.mla` (разные макеты столбцов): передай таблицы датологгера как `schema_table` и используй `log_raw(station, data)`. См. `DataloggerBuilder` и `tests/test_datalogger.py`; спецификация в NIC-MLA `DESIGN-MLA-datalogger.ru.md`.
