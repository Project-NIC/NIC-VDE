# Volkov Data

[English documentation here](README.md) | [Česká dokumentace zde](README_cs.md)

Кроссплатформенный двухпанельный файловый менеджер в стиле **Volkov Commander**,
написанный на Python с использованием **prompt_toolkit**. Просматривает локальную
файловую систему и умеет *заходить внутрь* контейнеров **NIC-MLA**, показывая
каждую записанную запись как файл.

> Статус: **1.0** — построено на **NIC-MLA v1.0**. Двухпанельный просмотр,
> файловые операции, просмотрщик файлов и записей, а также MLA-бэкенд для
> просмотра записей. MLA — намеренно *глупый* контейнер, поэтому бэкенд
> опирается на его самоописывающие таблицы: **таблица схемы** декодирует каждый
> упакованный payload в реальные значения + единицы, а **таблица станций**
> превращает 1-байтовый индекс станции из лога в реальные регион/номер — и то и
> другое сразу попадает в экспорт CSV/SQL.
>
> Архитектура повторяет философию самого MLA — **глупые библиотеки + тонкий
> клей**: внизу переиспользуемые библиотеки (`export` плюс встроенный MLA и его
> ридер), а бэкенды над ними — лишь тонкие адаптеры. Весь `volkov_core/` не
> зависит от GUI, поэтому пригоден и для использования без интерфейса.

## Запуск

```bash
pip install -r requirements.txt
python3 volkov_data.py [левый_каталог] [правый_каталог]
```

**Клавиши**

| Клавиша | Действие |
|---|---|
| `Tab` | переключить панель |
| `↑/↓ PgUp/PgDn Home/End` | перемещение курсора |
| `Enter` | открыть каталог / войти в `.mla` / на уровень вверх (`..`) |
| `F1` | сведения о выбранном элементе / записи |
| `F2` | проверка контейнера `.mla` — число корректных / мёртвых / повреждённых слотов |
| `F3` | просмотр содержимого файла или записи (текст/hex) |
| `F4` | просмотр записи с декодированными значениями и единицами (по схеме) |
| `F5` | копировать выбранный файл на другую панель |
| `F6` | переименовать или переместить — внутри `.mla` экспорт всех записей в CSV |
| `F7` | создать каталог |
| `F8` | удалить (с подтверждением) |
| `F9` | выпадающее меню (сортировка, язык, экспорт в SQL, …) |
| `F10` / `q` / `Ctrl-Q` | выход · `Esc` закрывает любое всплывающее окно |

Нажмите `Enter` на `samples/weather.mla`, чтобы войти внутрь и просмотреть его записи.

## Тесты

Логика в `volkov_core/` не зависит от GUI, поэтому покрыта набором тестов на
стандартной библиотеке `unittest` (без дополнительных зависимостей):

```bash
python3 -m unittest discover -s tests
```

Тесты создают временные MLA-контейнеры на лету, а также проверяют приложенный
образец `samples/weather.mla`.

## Структура

```
volkov_data.py           GUI на prompt_toolkit (тонкая оболочка над volkov_core)
volkov_core/             логика без GUI — пригодна для самостоятельного использования
  backend.py               абстракция бэкенда хранилища (Entry / Backend)
  local.py                 LocalBackend — файловая система хоста
  mla.py                   MlaBackend — тонкий адаптер: записи как «файлы»,
                           декодирование по схеме + разбор станции + экспорт
  export.py                глупая библиотека — обобщённые строки → CSV / SQLite
  stations.py              клей — индекс станции → реальные регион/номер
samples/make_sample.py   генератор самоописывающего образца метеостанции
samples/weather.mla      приложенный образец (упакованные строки + схема + станции)
tests/                   набор тестов unittest для volkov_core (без GUI)
third_party/nic_mla/     встроенный NIC-MLA v1.0 — канонический формат данных (Python + C)
  tools/mla_schema.py      host-only сборщики и ридеры схемы/станций (VDE их линкует)
docs/vc-reference/       один архив Volkov Commander (BSD-2) — резервный образец UI
```

Приложение читает формат логгера через эталонную Python-реализацию MLA
(`third_party/nic_mla/nic_mla.py`) и декодирует payload-ы и станции через
host-only ридер (`tools/mla_schema.py`), оба поддерживаются побайтово
идентичными ядру на C. Исходники Volkov Commander служат **только образцом
поведения** — это не портированный код.
