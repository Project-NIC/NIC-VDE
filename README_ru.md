Всё **семейство Arduino** NIC в одном месте — программная/информационная сторона
NIC. Каждый компонент сохраняет своё имя и свой README и остаётся пригодным к
самостоятельному использованию (drop-in, в стиле Arduino-библиотеки).

```
NIC-Arduino  (семейство Arduino)
├── mla/         основа — контейнер/формат лога NIC-MLA
├── dmd/         опционально: сжатие NIC-DMD
├── ksf/         опционально: шифрование NIC-KSF
├── glue-in/     клей: запись данных в лог MLA
├── glue-out/    клей: чтение / экспорт лога MLA (CSV, SQLite, …)
├── mseed/       сейсмо-экспорт: лог MLA → miniSEED (международный стандарт)
└── vde/         просмотрщик VDE (Volkov Data) — просмотр и экспорт логов MLA
```

## Как это складывается вместе

**MLA — это основа.** NIC-узел логирует отсчёты в контейнер NIC-MLA — это сердце
стороны данных, и оно самодостаточно. Оно намеренно простое.

Всё остальное **опционально, наслоено поверх MLA — бонус, а не требование:**

- **dmd/** — если нужно хранить отсчёты *сжатыми*, MLA умеет записывать их через
  NIC-DMD. Чистый MLA прекрасно работает без него.
- **ksf/** — если нужно хранить отсчёты *зашифрованными*, это делает NIC-KSF.
  Тоже опционально.
- **glue-in/ · glue-out/** — клей, который пишет в лог MLA и читает/экспортирует
  из него. Библиотеки — это детали; клей соединяет их под задачу.
- **mseed/** — сейсмо-экспорт. Сейсмограф NIC-Quake / NIC-Station сохраняет в
  MLA; когда нужен *международный* сейсмологический формат, mseed превращает этот
  лог MLA в miniSEED (ObsPy / SeisComp / FDSN). Создан для сейсмо-платформы; то,
  что его может использовать кто-то ещё, — бонус.
- **vde/** — просмотрщик VDE (Volkov Data): настольное приложение для просмотра и
  экспорта логов MLA.
